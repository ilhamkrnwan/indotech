# 003: WordPress Core Plugin (`indotech-core`)

Logika bisnis dan data PT Indotech Berkah Abadi diletakkan pada core plugin `wp-content/plugins/indotech-core/` untuk memastikan integritas data jika tema visual diganti.

## 1. Struktur Folder Plugin
```
indotech-core/
├── composer.json               # Dependensi Carbon Fields & autoload composer
├── vendor/                     # Folder vendor autogenerated oleh composer
├── indotech-core.php           # Entrypoint utama plugin
└── src/                        # Sumber berkas logika modular
    ├── Plugin.php              # Bootstrap utama inisialisasi plugin
    ├── PostTypes.php           # Registrasi brand, product, industry, application CPT
    ├── Taxonomies.php          # Registrasi product_cat, download_category taxonomies
    ├── MetaFields.php          # Konfigurasi Carbon Fields meta & Options Page
    ├── InquiryHandler.php      # Setup custom DB table, form AJAX handler, email SMTP
    ├── AdminDashboard.php      # Rendering WP_List_Table prospek leads di dashboard admin
    └── RestApi.php             # Registrasi custom rest endpoints indotech/v1
```

## 2. Inisialisasi Bootstrapper (`indotech-core.php`)
File entrypoint memanggil composer autoloader dan menginisialisasi bootstrap class:
```php
<?php
/**
 * Plugin Name: Indotech Core
 * Description: Core Business Logic, Custom Post Types, Custom Databases, and REST API for PT Indotech Berkah Abadi.
 * Version: 1.0.0
 * Author: PT Indotech Berkah Abadi
 * License: GPL2
 */

if (!defined('ABSPATH')) exit;

// Load Composer Autoloader
if (file_exists(dirname(__FILE__) . '/vendor/autoload.php')) {
    require_once dirname(__FILE__) . '/vendor/autoload.php';
}

// Inisialisasi Core Plugin
register_activation_hook(__FILE__, [ 'Indotech\\Core\\Plugin', 'activate' ]);
register_deactivation_hook(__FILE__, [ 'Indotech\\Core\\Plugin', 'deactivate' ]);

add_action('plugins_loaded', [ 'Indotech\\Core\\Plugin', 'get_instance' ]);
```

## 3. Activation Hook (`Plugin::activate`)
Pada saat plugin diaktifkan:
1.  Menjalankan `dbDelta` di `InquiryHandler` untuk membuat tabel `wp_indotech_inquiries`.
2.  Menjalankan pembersihan rule rewriting URL (`flush_rewrite_rules()`) agar URL custom post types (`/products/`, `/brands/`) langsung aktif tanpa perlu membuka halaman Permalinks di Dashboard secara manual.
